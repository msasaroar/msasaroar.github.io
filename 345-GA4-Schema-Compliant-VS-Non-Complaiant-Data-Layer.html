GA4 Data Layer Comparison

GA4-Compliant Data Layer

This is an example of a data layer structured to meet the GA4 schema requirements.

{
  "event": "purchase",
  "ecommerce": {
    "transaction_id": "T123456",
    "value": 299.99,
    "currency": "USD",
    "items": [
      {
        "item_name": "Laptop",
        "item_id": "LPT123",
        "price": 299.99,
        "quantity": 1,
        "category": "Electronics"
      }
    ]
  }
}

Explanation:

  • event: Defines the type of interaction (e.g., "purchase").
  • ecommerce: Contains transaction-specific details.
  • items: Each product in the transaction is represented as an object with required attributes like item_name, item_id, price, etc.

Non-Compliant Data Layer

This is an example of a data layer that does not follow the GA4 schema, leading to potential issues in tracking.

{
  "action": "transactionComplete",
  "orderDetails": {
    "orderID": "T654321",
    "totalAmount": 299.99,
    "currencyCode": "USD",
    "products": [
      {
        "productName": "Laptop",
        "productCode": "LPT123",
        "cost": 299.99,
        "units": 1,
        "categoryName": "Electronics"
      }
    ]
  }
}

Explanation:

  • action: Uses a non-standard key; GA4 expects event.
  • orderDetails: Does not use the ecommerce object required by GA4.
  • products: Uses incorrect attribute names like productName and cost instead of item_name and price.
