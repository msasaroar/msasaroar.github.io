Introduction

GA4 Schema refers to the structure and format of data used in Google Analytics 4 (GA4) for tracking events and user interactions on websites or apps. The schema ensures that data is organized effectively for analysis in GA4.

Components of GA4 Schema

1. Events

Events capture specific user actions or occurrences (e.g., page_view, purchase, add_to_cart). They are defined with an event name and may include additional parameters.

{
  "event_name": "purchase",
  "parameters": {
    "transaction_id": "T12345",
    "value": 59.99,
    "currency": "USD"
  }
}

2. Parameters

Key-value pairs that provide more context to the event. Examples:

  • value: Total value of a purchase.
  • currency: The currency code for a transaction.
  • item_name: Name of an item in an e-commerce transaction.

3. User Properties

Attributes of a user (e.g., country, preferred_language) that remain consistent across sessions and events.

{
  "user_properties": {
    "membership_tier": "gold"
  }
}

4. Items

A nested object for e-commerce events describing individual products or services related to the event.

{
  "event_name": "add_to_cart",
  "parameters": {
    "items": [
      {
        "item_id": "P001",
        "item_name": "T-Shirt",
        "price": 19.99,
        "quantity": 2
      }
    ]
  }
}

5. Reserved Parameters

GA4 reserves certain parameters for specific purposes, such as:

  • event_name: Name of the event.
  • value: Numeric value for monetary events.
  • currency: ISO currency code.

Why is GA4 Schema Important?

  • Consistent Data Collection: Ensures events and parameters are correctly captured for accurate analysis.
  • Flexibility: Custom parameters enable tracking of unique business metrics.
  • Enhanced Analysis: Proper structure allows advanced reporting and insights.
  • Compatibility with Tools: Integrates seamlessly with platforms like Google Ads and BigQuery.

Key Considerations for GA4 Schema

  • Follow GA4 naming conventions for events and parameters.
  • Use standard events (e.g., login, purchase) for better compatibility.
  • Avoid overloading events with unnecessary parameters.
  • Use a data layer to manage and structure event data before sending it to GA4.
