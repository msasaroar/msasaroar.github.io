02. Activities for setting up Server Side Conversion Tracking in Google Ads with GTM

1. Set Up a Server Container in GTM:
Go to Google Tag Manager.
Create a new Server container.
Deploy it on Google Cloud or another server.

2. Configure Google Cloud for GTM Server:
Create a new Google Cloud project.
Enable Cloud Run and App Engine.
Deploy the GTM server container.

3. Set Up a Custom Domain for the Server:
Configure a subdomain (e.g., sst.yourdomain.com).
Update DNS settings to point to the Cloud Run instance.

4. Modify Google Ads Conversion Tracking for Server-Side:
In Google Ads, create or retrieve the conversion action.
Note down the Conversion ID and Conversion Label.
Enable “Enhanced Conversions” if applicable.

5. Configure Google Ads Client in Server GTM:
Add a new Google Ads Conversion Tracking Client.
Ensure it listens for incoming conversion events.

6. Forward Conversion Events from Web GTM to Server GTM:
In Web GTM, modify conversion tracking tags.
Set the "Send to Server Container" option and use the custom domain.

7. Create a Google Ads Conversion Tag in Server GTM:
Set up a new Google Ads Conversion tag.
Use the Conversion ID and Conversion Label.
Map event data (e.g., transaction ID, revenue).

8. Implement User Identifiers for Better Match Rate:
Send hashed email, phone number, or first-party data.
Use SHA256 hashing for secure transmission.

9. Test and Validate Setup:
Use Google Tag Assistant and GTM Debug Mode.
Check conversion data in Google Ads.
Verify event data in the GA4 DebugView.

10. Optimize and Monitor Performance:
Regularly review conversion accuracy.
Optimize event parameters to improve match rates.
Monitor Cloud Run costs and performance.

03. Activities for setting up Server Side Conversion Tracking in Google Ads with GTM

1. Set Up a Server Container in GTM:

Create a new Server container.
Deploy it on Google Cloud or another server.

2. Configure Google Cloud for GTM Server:

Enable Cloud Run and App Engine.
Deploy the GTM server container.

3. Set Up a Custom Domain for the GTM Server:
Configure a subdomain (e.g., tracking.yourdomain.com).
Update DNS settings to point to the Cloud Run instance.

4. Get LinkedIn Insight Tag Information:
Log in to your LinkedIn Ads account.
Navigate to "Account Assets" → "Insight Tag."
Copy the Partner ID for tracking setup.

5. Configure the LinkedIn Client in Server GTM:
In Server GTM, create a new LinkedIn Insight Tag Client.
Ensure it listens for relevant conversion events.

6. Modify Web GTM Tags for Server-Side Tracking:
In Web GTM, modify LinkedIn Insight Tag.
Update settings to send events to the server container instead of LinkedIn directly.

7. Set Up LinkedIn Conversion Tag in Server GTM:
In Server GTM, create a LinkedIn Insight Conversion Tag.
Use the Partner ID and map conversion events.
Ensure correct event parameters like email, conversion value, and currency.

8. Implement User Identifiers for Better Attribution:
Send hashed email (SHA256) for better match rates.
Ensure data privacy compliance (GDPR, CCPA).

9. Test and Debug:
Use GTM Preview Mode to check data flow.
Validate conversions using LinkedIn’s Tag Tester Tool.
Monitor LinkedIn's Matched Audiences & Conversions.

10. Optimize and Monitor:
Regularly check LinkedIn Campaign Manager for accuracy.
Adjust event mapping and parameters to improve match rates.
Monitor Google Cloud costs and optimize performance.

04. Activities for setting up Server Side Conversion Tracking in Meta Pixel with GTM

Update DNS settings to point to the Cloud Run instance.

4. Get Meta Pixel Information:
Log in to Facebook Business Manager.
Navigate to Events Manager → Meta Pixel.
Copy the Pixel ID for tracking setup.

5. Set Up Meta Conversions API (CAPI):
In Events Manager, go to the Conversions API setup.
Select Google Tag Manager (Server-Side) as the integration method.

6. Configure the Meta Client in Server GTM:
In Server GTM, create a new Meta Pixel Client.
Ensure it listens for relevant conversion events.

7. Modify Web GTM Tags for Server-Side Tracking:
In Web GTM, modify Meta Pixel Tags.
Update settings to send events to the server container instead of Meta directly.

8. Create a Meta Conversion API Tag in Server GTM:
In Server GTM, create a Meta Conversions API Tag.
Use the Pixel ID and configure event mapping.
Ensure correct parameters like event name, transaction ID, and purchase value.

9. Implement User Identifiers for Better Attribution:
Send hashed user data (SHA256) like email, phone number, and IP address.
Ensure data privacy compliance (GDPR, CCPA).

10. Enable Event Deduplication:
Use the same Event Name and Event ID for browser and server events.
Ensure Meta correctly deduplicates the events.

11. Test and Debug:
Use GTM Preview Mode to check data flow.
Validate events in Meta Events Manager Debugging Tool.
Monitor Event Match Quality Score.

12. Optimize and Monitor:
Regularly check Meta Business Manager for conversion accuracy.
Adjust event mapping and parameters to improve match rates.
Monitor Google Cloud costs and optimize performance.

Go to Google Tag Manager.

4. Enable Shopify Web Pixel API:
Log in to Shopify Admin.
Go to Settings → Customer Events.
Click Manage Pixels and create a Custom Pixel.
Get the Shopify Pixel API Key.

5. Implement Web GTM in Shopify:
Go to Online Store → Themes → Edit Code.
Add the GTM Web Container gtag.js script in `theme.liquid`.

6. Set Up Shopify Web Events:
Use Shopify Checkout Extensibility to track events like Page View, Add to Cart, Purchase.
Ensure event parameters like order ID, value, and currency are included.

7. Modify Web GTM Tags for Server-Side Tracking:
In Web GTM, modify Shopify event tags (e.g., Purchase, Add to Cart).
Update settings to send events to the server container.

8. Set Up a Shopify Conversion API Tag in Server GTM:
In Server GTM, create a Shopify Conversion API Tag.
Use the Shopify Pixel API Key and send server events to Shopify.
Ensure parameters like transaction ID, revenue, and currency are mapped.

9. Implement User Identifiers for Better Attribution:
Send hashed user data (SHA256) like email, phone number, and IP address.
Ensure compliance with GDPR, CCPA, and Shopify’s privacy policies.

10. Enable Event Deduplication:
Use the same Event Name and Event ID for browser and server events.
Ensure Shopify correctly deduplicates the events.

11. Test and Debug:
Use GTM Preview Mode to check data flow.
Validate events in Shopify Customer Events Debugging Tool.
Check Shopify Analytics to confirm data accuracy.

12. Optimize and Monitor:
Regularly check Shopify reports for conversion accuracy.
Adjust event mapping and parameters to improve match rates.
Monitor Google Cloud costs and optimize performance.
