Understanding Custom Events in GA4

What Are Custom Events?

In Google Analytics 4 (GA4), custom events are actions you define to track specific user interactions that GA4 doesn’t track by default. Unlike Universal Analytics, GA4 is event-based, so everything — from a page view to a video play — is an event.

Why Use Custom Events?

You use custom events when you want to track something unique to your website or app — like when a user clicks a "Download CV" button, submits a form, or scrolls past a certain point. GA4’s default events don’t cover these interactions, so custom events let you fill that gap.

How to Set Up a Custom Event with Google Tag Manager

  1. Add an id to your HTML element (e.g., <button id="download-cv-btn">).
  2. Enable click variables in GTM (like Click ID, Click Text).
  3. Create a trigger: Choose "All Elements" > "Some Clicks" > Set Click ID = download-cv-btn.
  4. Create a GA4 Event Tag:
    • Event Name: download_cv
    • Optional parameters like: button_text: {{Click Text}}
  5. Preview, test, then publish your container.

Track Conversions

After your custom event is live, you can mark it as a conversion in GA4. Go to Admin > Events, find your event name, and toggle the conversion switch.

Conclusion

Custom events in GA4 give you control over what you track, so you can collect data that matters to your business. They are powerful for tailoring your analytics setup to your specific goals.
