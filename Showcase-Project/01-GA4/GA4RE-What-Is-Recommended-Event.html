What is a Recommended Event in GA4?

Definition

Recommended events in Google Analytics 4 (GA4) are events that Google suggests you implement to get the most out of GA4’s advanced reporting and features. These events aren’t tracked automatically, but if your site or app includes relevant actions, Google recommends you set them up using specific event names and parameters.

Why Use Recommended Events?

  • They unlock enhanced reporting in GA4 (e.g., eCommerce, media tracking).
  • They follow naming best practices recognized by Google Analytics.
  • They are future-proof and more likely to integrate with new GA features.

Examples of Recommended Events

  • sign_up — Track when a user registers for an account.
  • login — Track user logins.
  • purchase — Track completed transactions.
  • view_item — Track product detail views.
  • add_to_cart — Track when a product is added to a cart.

How to Implement

You can use Google Tag Manager or gtag.js to send these events. Here's an example of how to send a "sign_up" event using gtag: 


gtag('event', 'sign_up', {
  method: 'Google'
});

Where to Find the List

Google provides a full list of recommended events in their official documentation.

Conclusion

Recommended events in GA4 give you a structured way to capture meaningful user interactions using standardized event names. If your site or app uses common features like sign-ups, purchases, or media views, following Google’s recommended event naming is a best practice.
