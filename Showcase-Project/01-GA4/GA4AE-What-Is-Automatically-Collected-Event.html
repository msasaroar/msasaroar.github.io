GA4 | Automatically Collected Events and Enhanced Measurement Events

Introduction

Google Analytics 4 (GA4) offers a robust system for tracking user interactions on your website or app using events. Events are essential for understanding user behavior and engagement. GA4 provides two types of events that are automatically collected when you set up the GA4 tracking code: Automatically Collected Events and Enhanced Measurement Events.

In this article, we will explore both types of events, how they are tracked, and how you can use them to gain deeper insights into user activity on your site.

Automatically Collected Events

These are the default events tracked by GA4 as soon as you set up the GA4 tag on your website. They require no additional configuration and are automatically collected by the system.

These events are essential for basic user and session-level tracking.

Enhanced Measurement Events

Enhanced Measurement is a feature within GA4 that allows you to track more granular interactions with users on your site, such as scrolling, outbound link clicks, video interactions, and file downloads. These events are also collected automatically, but they can be enabled or disabled in the GA4 admin panel depending on your needs.

Enhanced Measurement events offer deeper insights into user behavior beyond basic interactions.

Table: Automatically Collected Events vs Enhanced Measurement Events

Below is a comparison of the events that are automatically collected by GA4, distinguishing between those that are part of Enhanced Measurement and those that are not. A short description is also included for each event to help you understand its purpose.

Event Name Enhanced Measurement? Triggered When... Short Description
01. page_view ✅ Yes A page is viewed or the browser history changes. Tracks when a user views a page on your site.
02. scroll ✅ Yes A user scrolls to 90% of the page height. Measures how far users scroll down the page, indicating content engagement.
03. click ✅ Yes A user clicks an outbound link (link to another domain). Tracks clicks on external links leading away from your website.
04. file_download ✅ Yes A user clicks to download a file. Monitors when users download files such as PDFs or documents.
05. video_start ✅ Yes A user starts an embedded video (YouTube, etc.). Tracks when a user begins watching an embedded video.
06. video_progress ✅ Yes A user reaches 10%, 25%, 50%, or 75% progress in a video. Monitors user engagement with video content as they progress through it.
07. video_complete ✅ Yes A user finishes watching the video. Tracks when users watch the entire video.
08. form_start ✅ Yes A user starts filling out a form. Tracks when a user begins to interact with a form (e.g., sign-up, contact form).
09. form_submit ✅ Yes A user submits a form. Records when a user submits a form on your website.
10. first_visit ❌ No A user visits the site for the first time. Indicates when a user is visiting your site for the very first time.
11. session_start ❌ No A new session starts (usually after 30 minutes of inactivity). Tracks when a new session begins after a period of user inactivity.
12. user_engagement ❌ No A user engages with the site for at least 10 seconds or other actions. Measures user engagement based on time spent or actions taken on your site.

Conclusion

In summary, GA4 automatically collects both basic and enhanced user interaction data through its system of events. Automatically Collected Events track fundamental user actions, such as page views, sessions, and first visits, while Enhanced Measurement Events offer more detailed insights into how users engage with content, such as scrolling, file downloads, and video progress.

By understanding the difference between these two types of events, you can configure your GA4 setup to capture the data most relevant to your business goals and user experience analysis.
