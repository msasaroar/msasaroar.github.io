Google Tag Manager Built-In Variables

A guide to configure, explain, and demonstrate GTM Built-In Variables.

1. Page URL Variable

The Page URL variable captures the full URL of the page, including the protocol, hostname, path, and query string.

Configuration Steps [ Click ]

  1. Go to GTM > Variables > Configure.
  2. Enable the built-in variable: Page URL.

Demonstration:

https://example.com/products?category=shoes

2. Page Hostname Variable

The Page Hostname variable captures the hostname portion of the URL (e.g., example.com).

Configuration Steps [ Click ]

  1. Go to GTM > Variables > Configure.
  2. Enable the built-in variable: Page Hostname.

Demonstration:

example.com

3. Page Path Variable

Captures the path portion of the URL (e.g., /products).

Configuration Steps [ Click ]

  1. Go to GTM > Variables > Configure.
  2. Enable the built-in variable: Page Path.

Demonstration:

/products

4. Click URL Variable

Captures the URL of the clicked link.

Configuration Steps [ Click ]

  1. Go to GTM > Variables > Configure.
  2. Enable the built-in variable: Click URL.

Demonstration:

https://example.com/signup

5. Click Text Variable

The Click Text variable captures the text content of the clicked element. It is commonly used to track user interactions like button clicks, link clicks, or specific elements on the page.

Configuration Steps [ Click ]

  1. Go to GTM > Variables > Configure.
  2. Enable the built-in variable: Click Text.

Demonstration:

Clicked Element Text: "Buy Now"

6. Click Classes Variable

The Click Classes variable retrieves the CSS classes of the clicked element, making it useful for tracking specific user interactions on your website.

Configuration Steps [ Click ]

  1. Go to GTM > Variables > Configure.
  2. Enable the built-in variable: Click Classes.

Demonstration:

btn-primary btn-add-to-cart

7. Click ID Variable

The Click ID variable captures the ID attribute of the clicked element on the page, allowing you to track interactions with specific elements.

  1. Go to GTM > Variables > Configure.

15. Event Name Variable

The Event Name variable captures the name of the event triggered in GTM, such as a click or a page view.

  2. Enable the built-in variable: Event Name.

16. First Party Cookie Variable

The First Party Cookie variable captures the value of a specified cookie set by your website.

Configuration Steps [ Click ]

  1. Go to GTM > Variables > New > First Party Cookie.
  2. Configure the cookie name and other details as needed.

Demonstration:

user_id=12345

17. Custom JavaScript Variable

Allows you to write custom JavaScript to create advanced variables. The custom JavaScript code must return a value.

Configuration Steps [ Click ]

  1. Go to GTM > Variables > New > Variable Configuration.
  2. Select Custom JavaScript.
  3. Add your custom JavaScript code.

Example Code:

  function() {
    return document.title; // Returns the current page title
  }

18. Data Layer Variable

The Data Layer variable captures a value from the data layer based on the specified key.

Configuration Steps [ Click ]

  1. Go to GTM > Variables > New > Data Layer Variable.
  2. Specify the data layer key to extract the value.

Demonstration:

{"product_id": "12345"}

19. JavaScript Variable

The JavaScript variable captures a value from custom JavaScript code executed on the page.

Configuration Steps [ Click ]

  1. Go to GTM > Variables > New > JavaScript Variable.
  2. Enter the custom JavaScript code to define the variable's value.

Demonstration:

function() { return document.title; }

20. Environment Name Variable

The Environment Name variable captures the environment (e.g., Development, Testing, or Live) used in GTM.

Configuration Steps [ Click ]

  1. Ensure environments are set up in GTM (Admin > Environments).
  2. The Environment Name is available in GTM by default.

Demonstration:

Live
